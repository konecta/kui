kGrid

Grilla para aplicaciones web de Konecta SA.

Ejemplos

Requerimientos

  • Jquery 1.11.1 o superior
  • Bootstrap 3.x
  • Font Awesome 4.x

Parametros de .kGrid()

Requeridos

url
URL para la peticion POST del listar objetos de la clase.
id
Nombre del atributo que sera considerado como identificador unico o Primary Key de la clase.
campos
Array de campos de la entidad, con el siguiente formato: 
[{
   nombre: '',          // (requerido) nombre del atributo
   titulo: '',
   tipo: '',            // Tipo de campo: encabezado, destacado o score
   ancho: '',			// Solo se utiliza en kGrid Simple.
   atributos: [{		// array de atributos para el input 
                   atributo: '',
                   valor: ''
               },...],
   formato: function(){}// Función para mostrar algo distinto a lo recibido
},...]

Opcionales

tarjetas
Presentación de los datos con apariencia de tarjetas. Por defecto es false.
ajax
Tipo de peticion, por defecto es GET.
jqGrid
Compatibilidad con llamadas preparadas para jqGrid, por defecto false
data
Parametros a ser enviados en la peticion POST. Sus valores por defecto son: {}
permisos
Funciones que se llaman para editar, activar y remover objetos. Sus valores por defecto son: 
{
   editar: null,
   activar: null,
   remover: null
}
botones
Lista de objetos para incluir botones adicionales para cada row, formato: 
[
	{
		comentario: '',                         // (requerido) Texto para el tooltip
		icono: '',				// (requerido) tipo de bootstrap icon
		enlace: '',				// href o enlace del a, por defecto Javascript:void(0);
		mostrar: function(item){},                  // funcion que define cuando muestrar el boton
		onclick: function(item){},                  // funcion onclick, recibe como argumento al item
		atributos: [
						{			// array de atributos para el a 
							atributo: '',
							valor: ''
						},...
				   ]
   	},...
]
estado
Funcion que debe retornar true cuando el objeto sera mostrado como activo, y false cuando sera identificado como un objeto removido.
retorno
Nombre de los campo en la respuesta del POST Sus valores por defecto son: 
{
   lista: 'lista',                     // Lista de objetos
   pagina: 'pagina',                   // Número de página actual
   totalPaginas: 'totalPaginas',       // Total de páginas
   totalDatos: 'totalDatos',           // Total de objetos en todas las páginas juntas 
}
loadComplete
Funcion adicional que se llama al terminar la carga de los objetos.
paginador
id del div que se utilizará como paginador.
onclick
Se aplica cuando se hace click en el row si es un string se asume que es una url, también puede ser una función
ondblclick
Función que se aplica cuando se hace doble click en el row. Sólo puede ser utilizado cuando no hay la orden "onclick".
