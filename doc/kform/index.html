kForm

Formulario para aplicaciones web de kUI.

El proyecto kForm es una librería javascript que genera formularios html en base a una serie de definiciones.

Se centra en la implementación rápida de formularios. Se encuentra en las primera etapa de desarrollo.

Parámetros

Requeridos

Campo Descripción Valor
submit Función o servicio que se ejecutará al guardar o finalizar (submit) el formulario. Acepta dos tipos de valores:
  • String. Sería la url al servicio (por defecto POST) al que se llamará enviando como atributos los campos visibles en el formulario.
  • Function. Una función que recibe como argumento el formulario serializado, es decir un objeto que tiene como atributos los campos visibles del formulario.
campos Array de campos que contendrá el formulario. 
[{
   nombre: '',              // (requerido) nombre del atributo
   titulo: '',              // Es opcional, por defecto toma el valor de campo.nombre
   placeholder: '',         // Es opcional, por defecto toma el valor de campo.titulo
   mensaje: '',             // Es opcional, se utiliza en el tooltip de las validaciones
   requerido: false,        // Es opcional, sirve para validar un campo como requerido
   icono: ''                // Es opcional, determina el ícono que se mostrará delante 
                            // del input, debe ser una clase de Font-awesome, pero sin 
                            // el prefijo "fa-", por ejemplo "at" en lugar de "fa-at"
   tipo: '',                /* Es opcional, y acepta los siguientes valores:
                             * - texto (es el tipo por defecto)
                             * - booleano
                             * - numero (enteros)
                             * - decimal 
                             * - archivo
                             * - combo
                             * - fecha
                             * - hora
                             * - fecha-hora
                             *
                             */
   opciones: {            // Requerido cuando tipo es "combo", en los otros tipos no tiene utilidad.                            
      origen: '',         // Requerido, puede ser una URL a un servicio o un array con objetos.
      id: '',             // Requerido, se utiliza para definir el atributo "value" de los <option>
      formato: ''         // Requerido, puede ser un string con el nombre del campo que se 
                          // utilizará como dato visible, o  una función que recibirá como
                          // argumento un objeto de la lista y en base al mismo podrás 
                          // formatear la salida visible
      ajax: ''            // Opcional, por defecto GET. Sólo sirve cuando origen es string.
      data (opcional, por defecto { // Parámetros que se enviarán al servicio origen, 
                                    // solo tiene utilidad cuando origen es un string
          _search: false,
          filters: null,
          page:    1,
          rows:    10,
          sidx:    /* acá va el valor de campo.opciones.id */,
          sord:    asc,
          todos:   true,
      })
   }
},...]

Opcionales

Campo Descripción Valor por defecto
origen Campo que define los datos iniciales del formulario, cuando se omite se considera un formulario de creación. Acepta dos tipos de valores:
  • String. URL del servicio (GET por defecto).
  • Object. Los datos a utilizar en el formulario.
 {}
ajaxOrigen Tipo de petición para el servicio de origen. Sólo se utiliza cuando origen es del tipo string. "GET"
dataOrigen Parámetros a ser enviados en la petición AJAX del servicio de origen. Sólo se utiliza cuando origen es del tipo string. {}
soloLectura Campo que permite la creación de formularios de solo lectura (fieldset=disabled). false
ajaxSubmit Tipo de petición para el servicio de submit. Sólo se utiliza cuando submit es del tipo string. "POST"
botonSubmit id del botón (o clase de los botones) que se utilizarán para hacer submit al formulario. En caso de omisión kGrid crea un botón debajo del formulario.
loadComplete Función adicional que se llama al terminar la carga de los objetos.
